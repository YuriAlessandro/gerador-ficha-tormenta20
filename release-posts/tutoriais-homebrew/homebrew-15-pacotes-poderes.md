## Bloco 1

**Título do bloco:** O que é um Pacote de Poderes

Enquanto a classe traz poderes **dela**, um **Pacote de Poderes** publica poderes avulsos — até **20** — que ficam disponíveis conforme a categoria. É como você adiciona uma leva de poderes de combate novos, ou poderes concedidos por uma divindade caseira, sem precisar criar uma classe inteira.

Comece em **Meus Homebrews → Novo homebrew → Novo Pacote de Poderes** (*"Novo Pacote de Poderes homebrew"*). Após o envelope, vem a lista com o cabeçalho **"Poderes (X / 20)"**.

---

## Bloco 2

**Título do bloco:** Geral ou de classe?

Clique em **"Adicionar poder"** (limite avisado em 20). A primeira decisão de cada poder é o campo **Tipo**:

- **Poder geral** — disponível conforme a **Categoria**. As categorias são: **Combate**, **Destino**, **Magia**, **Concedidos**, **Tormenta** e **Raça**.
- **Poder de classe** — aparece para quem joga uma classe específica. Surge o campo **Classe-alvo** (*"O poder aparece para quem joga essa classe."*).

Use "geral" para poderes que qualquer personagem elegível pode pegar (poderes de combate, por exemplo); use "de classe" para reforçar o repertório de uma classe específica (oficial ou homebrew).

## Bloco 3

**Título do bloco:** O editor de poder, de novo

Depois de escolher o tipo (e a categoria/classe-alvo), você cai no **mesmo editor de poder** dos poderes de classe — aquele do [post de Classes II](/blog/homebrew-classes-2-poderes). Ou seja, cada poder do pacote tem:

- **Nome** e **Descrição**, **Custo de PM (opcional)**, **"Pode repetir"**.
- Acordeões: **Pré-requisitos**, **Bônus passivos**, **Efeitos ativos**, **Arma natural**, **Rolagens** e **Conceder magias** (avançado).

Tudo que você aprendeu sobre bônus, requisitos e efeitos ativos se aplica igualzinho. Não há nada novo a aprender — é reaproveitamento puro.

## Bloco 4

**Título do bloco:** Quando usar pacote de poderes vs. classe

Uma dúvida comum: "isso devia ser um pacote de poderes ou parte de uma classe?". O critério:

- Se os poderes **definem uma forma de jogar completa** (com PV/PM, perícias, progressão), é **classe**.
- Se são **acréscimos** ao ecossistema existente — uma leva de poderes de combate, poderes concedidos de um deus novo, talentos de um estilo — é **pacote de poderes**.

Pacotes são ótimos para conteúdo modular: o usuário ativa o seu pacote de "Poderes de Arquearia Avançada" e pronto, eles aparecem para os personagens elegíveis, sem mexer em nenhuma classe.

Falta um tipo: o que **agrupa** vários homebrews seus numa publicação só. É o próximo post: [Coleções](/blog/homebrew-colecoes).
